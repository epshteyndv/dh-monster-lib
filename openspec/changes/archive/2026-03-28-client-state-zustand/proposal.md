## Why

Состояние каталога (загрузка, ошибка, список монстров) и **энкаунтера** сейчас размазано по `useState` / `useReducer` в `App.tsx`. По мере роста UI это усложняет тестирование, подписки и переиспользование логики. **Zustand** даёт лёгкий предсказуемый стор без лишнего бойлерплейта и хорошо сочетается с React 18 и TypeScript.

## What Changes

- Добавить зависимость **`zustand`**.
- Вынести логику энкаунтера и загрузки каталога в **типизированные Zustand-стор(а)**; `App` и при необходимости другие компоненты читают состояние и вызывают действия через хуки.
- Сохранить текущее **поведение** (энкаунтер, фокус, дубликаты, загрузка JSON) — **BREAKING** только для внутренней структуры кода, не для формата `monsters.json` и не для пользовательского сценария.

## Capabilities

### New Capabilities

- `client-state`: нормативные требования к тому, что клиентское состояние каталога и энкаунтера ведётся через Zustand с типизацией TypeScript.

### Modified Capabilities

_(none — поведение UI остаётся под `github-pages-monster-ui`; меняется только способ хранения состояния.)_

## Impact

- `package.json` / lockfile, новый модуль(и) стора в `src/`, рефакторинг `App.tsx`, возможное сокращение или перенос `encounterState.ts` (типы и чистая логика переходов).
