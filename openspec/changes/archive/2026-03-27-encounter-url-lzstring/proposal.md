## Why

Сериализация энкаунтера в query через JSON + Base64 URL-safe даёт длинные ссылки при большом числе монстров. Сжатие **LZString** (типичный вывод `compressToEncodedURIComponent`) укорачивает строку в URL, сохраняя тот же логический формат (версионированный JSON). Обратная совместимость со старыми ссылками **не требуется** — допустимо сменить кодек целиком.

## What Changes

- **BREAKING** (намеренно): старые ссылки с прежним `enc` (Base64 URL) перестают восстанавливать энкаунтер; новый формат заменяет их.
- Зависимость **`lz-string`** в клиентском приложении; кодирование/декодирование в `encounterUrlCodec` (или эквивалент) переводится на **LZString** вместо ручного UTF-8 + Base64 URL-safe.
- Требования в спецификациях **github-pages-monster-ui** и **client-state** обновляются под описание сжатого представления, без обязательства читать старый формат.

## Capabilities

### New Capabilities

_(none)_

### Modified Capabilities

- `github-pages-monster-ui`: требование **Encounter state in URL** — описать хранение в `enc` как сжатую строку через **LZString** (не Base64 URL-safe вручную).
- `client-state`: требование **Encounter URL codec in client code** — явно указать использование библиотеки **lz-string** (или аналог с тем же контрактом) для сжатия/распаковки полезной нагрузки перед помещением в query.

## Impact

- `package.json` + lockfile; модуль `encounterUrlCodec` (и связанный sync); возможное переименование/константа версии wire-format для явного разрыва со старыми ссылками.
