## Context

Сейчас энкаунтер кодируется как JSON → UTF-8 → Base64 URL-safe в параметре `enc`. Пользователь хочет **короче** в URL; без сохранения совместимости со старыми ссылками.

## Goals / Non-Goals

**Goals:**

- Использовать **LZString** из пакета **`lz-string`** (стандартная связка для браузера: `compressToEncodedURIComponent` / `decompressFromEncodedURIComponent`), чтобы строка в query была **URL-safe** и обычно короче, чем несжатый Base64-подобный вариант для повторяющихся id.
- Сохранить **версионированный JSON** как логическое тело (например `{ "v": 1, "ids": [...] }`), затем сжать целиком; при смене формата поднять **версию** `v` в payload.
- **Не** поддерживать декодирование старых Base64 URL ссылок.

**Non-Goals:**

- Синхронизация с `localStorage`, другие query-параметры.
- Выбор алгоритма сжатия кроме LZString/lz-string.

## Decisions

1. **Библиотека:** `lz-string` из npm — широко используется, без `Buffer`, подходит для Vite/GitHub Pages.

2. **API:** `LZString.compressToEncodedURIComponent(jsonString)` / `decompressFromEncodedURIComponent(enc)` — уже пригодно для подстановки в `URLSearchParams` без ручной base64url-обвязки.

3. **Версия wire-format:** увеличить константу `ENCOUNTER_URL_VERSION` (или эквивалент) при смене кодека, чтобы будущие расширения оставались явными; старые ссылки не обрабатывать.

4. **Лимит длины:** сохранить существующий ограничитель длины параметра (или скорректировать число после замера); при превышении — прежнее поведение (предупреждение, не писать).

## Risks / Trade-offs

- **[Risk]** Дополнительный вес бандла — **Mitigation:** маленькая библиотека; при необходимости tree-shaking только используемых функций.
- **[Risk]** Старые закладки с `enc` перестанут работать — **Mitigation:** принято пользователем (без миграции).

## Migration Plan

Деплой один раз; пользователи получают новые ссылки после изменения энкаунтера. Документировать в README при желании одной строкой.
