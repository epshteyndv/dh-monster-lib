## Why

Текущие записи монстров в YAML допускают произвольные поля (`description`, `tags`), что не совпадает с целевой **карточкой монстра** в стиле Daggerheart (пример: заголовок с тиром, мотивы, блок статов, опыт, список фич с типами). Нужна **единая схема данных** и отображение, чтобы каталог был предсказуемым для контрибьюторов и читабельным на сайте.

## What Changes

- Заменить «минимум `id` + `name` + произвольные поля» на **структурированную модель** карточки: `tier`, `flavor`, `motives`, объект **`stats`** (включая первичную атаку), строка **`experience`**, массив **`features`** с типом (passive / action / reaction) и телом текста.
- Обновить **пример данных** в `data/monsters.yaml` (и при необходимости пример в README) под новую схему.
- **Расширить** скрипт сборки каталога: валидация обязательных полей и типов (или явные ошибки при отсутствии секций).
- **Обновить UI** (React + Mantine): вёрстка карточки по секциям (шапка, блок статов, опыт, фичи), а не плоский список ключей.
