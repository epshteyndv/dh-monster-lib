## Why

Текущий UI на чистом TypeScript и ручном DOM сложнее поддерживать по мере роста каталога и экранов. **React** с типизацией и готовой **Mantine** даёт предсказуемую композицию компонентов, доступность и единый визуальный слой без написания низкоуровневой вёрстки с нуля.

## What Changes

- Заменить точку входа Vite на **React** (TSX): корневой рендер через `createRoot`, структура компонентов вместо императивного DOM.
- Подключить **Mantine** (`@mantine/core` и связанные пакеты по необходимости: стили, хуки): обёртка **`MantineProvider`**, использование компонентов библиотеки для списка монстров, области деталей и состояний загрузки/ошибки.
- Обновить конфигурацию сборки: **`@vitejs/plugin-react`**, PostCSS при необходимости для Mantine v7+, импорт стилей Mantine.
- **BREAKING** для структуры `src/`: удаление/замена `main.ts` + vanilla CSS на React-дерево и стили Mantine (поведение для пользователя каталога сохраняется).

## Capabilities

### New Capabilities

- `mantine-react-stack`: Требования к стеку React + TypeScript + Mantine: зависимости, провайдер темы, точка входа, отсутствие смешения с прежним vanilla UI.

### Modified Capabilities

- `github-pages-monster-ui`: Дополнительные требования к реализации интерфейса (React/TSX и Mantine) при сохранении существующего поведения (статика, base path, список/детали, ошибки загрузки).

## Impact

- `package.json` / `package-lock.json` — новые зависимости React, Mantine, плагин Vite.
- `vite.config.ts`, `index.html`, `src/**` — перестройка под React.
- `tsconfig.json` — поддержка JSX (например `jsx: react-jsx`).
- CI без изменений сути (по-прежнему `npm run build`), возможно дольше `npm ci` из‑за объёма зависимостей.
