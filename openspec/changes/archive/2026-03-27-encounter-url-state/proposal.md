## Why

Состояние энкаунтера живёт только в памяти: при обновлении страницы список монстров пропадает, ссылку на собранную встречу нельзя отправить другому человеку. Хранение снимка энкаунтера **в URL** даёт восстановление после перезагрузки и **шаринг** одной ссылкой без бэкенда.

## What Changes

- После загрузки каталога приложение **читает** закодированное состояние из URL (один query-параметр), **восстанавливает** энкаунтер (порядок и повторы по `id` монстра из каталога).
- При изменении энкаунтера приложение **обновляет** URL (без лишних записей в истории — `replaceState`), используя **расширяемый** формат полезной нагрузки (JSON) и кодирование **Base64 URL-safe** (`base64url`), чтобы строка безопасно жила в query и при необходимости расширялась новыми полями версии.

## Capabilities

### New Capabilities

_(none)_

### Modified Capabilities

- `github-pages-monster-ui`: требования к синхронизации энкаунтера с URL, восстановлению и шарингу.
- `client-state`: нормативно зафиксировать, что сериализация/парсинг энкаунтера в URL выполняется в клиентском коде (Zustand + history API / `window.location`).

## Impact

- Новый модуль кодирования/декодирования, подписка на стор энкаунтера и инициализация из `URLSearchParams`, аккуратная работа с Vite `BASE_URL` и длиной URL.
