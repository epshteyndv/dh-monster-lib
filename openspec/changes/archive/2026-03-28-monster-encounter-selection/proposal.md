## Why

Гейммастерам нужно собирать **встречу** из нескольких монстров из каталога. Сейчас UI ведёт себя как одиночный выбор: клик по строке каталога подменяет карточку, без накопления группы и без возможности добавить одного и того же монстра несколько раз (например, рой или два одинаковых соло).

## What Changes

- Вводится клиентское состояние **encounter** (энкаунтер): упорядоченный список **экземпляров** — каждый элемент ссылается на данные монстра из каталога.
- Клик по монстру в **списке каталога** **добавляет** новый экземпляр в энкаунтер (не заменяет предыдущий выбор).
- **Один и тот же** монстр по `id` из каталога может входить в энкаунтер **несколько раз**; экземпляры различаются **идентификатором записи** (не только `id` монстра).
- UI показывает энкаунтер отдельно от каталога и позволяет просматривать карточку для выбранного в энкаунтере экземпляра (детали в `design.md`).
