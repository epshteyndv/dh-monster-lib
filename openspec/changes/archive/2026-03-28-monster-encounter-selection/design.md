## Context

Сейчас `App` хранит `selected: Monster | null` и при клике по каталогу вызывает `setSelected(m)`. Каталог приходит из `loadCatalog()` как массив `Monster[]`.

## Goals / Non-Goals

**Goals:**

- Состояние **`encounter`**: массив записей `{ instanceId: string; monster: Monster }`, порядок = порядок добавления.
- Клик по строке каталога: **`append`** новой записи с новым `instanceId` (например `crypto.randomUUID()`).
- Отрисовка **списка энкаунтера** (имя + строка тира, как в каталоге) с возможностью **сфокусировать** запись для колонки с `MonsterCard`.
- Удаление одной записи из энкаунтера (кнопка «убрать»), чтобы список не был только растущим.

**Non-Goals:**

- Сохранение энкаунтера в `localStorage` или URL (только сессия).
- Изменение схемы данных каталога.

## Decisions

1. **Идентификатор экземпляра** — `instanceId: string` (UUID v4 в браузере). Один `monster.id` может повторяться в разных записях с разными `instanceId`.

2. **Фокус карточки** — `focusedInstanceId: string | null`. Клик по записи энкаунтера выставляет фокус. После **добавления** из каталога фокус переводится на **последнюю** добавленную запись. Если запись удалена и была в фокусе — фокус на соседнюю или `null`.

3. **Вёрстка** — три зоны: каталог | энкаунтер | карточка (например `Grid` 12 колонок: каталог `sm={4}`, энкаунтер `sm={3}`, карточка `sm={5}`) или стек на `base`; детали подгоняются под Mantine.

## Risks / Trade-offs

- **[Risk]** Узкая колонка на мобильном — **Mitigation:** вертикальный стек, порядок: каталог → энкаунтер → карточка.
