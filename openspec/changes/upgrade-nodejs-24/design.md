## Context

Проект — Vite + npm, деплой через GitHub Actions. Сейчас в workflow указан Node **20**. Запрос: перейти на **Node 24** из‑за политики поддержки версий на стороне GitHub.

## Goals / Non-Goals

**Goals:**

- Единая целевая версия **Node.js 24** в CI и в документации для разработчиков.
- Минимальный diff: версия в workflow + явное объявление в репозитории (`engines`, `.nvmrc`).

**Non-Goals:**

- Смена менеджера пакетов (pnpm/yarn), обновление мажорных зависимостей без необходимости.
- Поддержка нескольких версий Node в одном репозитории.

## Decisions

| Решение | Выбор | Rationale | Альтернативы |
|--------|--------|-----------|--------------|
| Версия в CI | `node-version: "24"` в `setup-node` | Соответствует запросу и актуальной линейке | `"24.x"` — менее предсказуемо при патчах |
| Фиксация для локальной разработки | `.nvmrc` с `24` + `engines.node: ">=24"` или `24.x` | nvm/fnm читают `.nvmrc`; `engines` даёт предупреждение при несовпадении | Только README — легче рассинхронить |
| Пересборка lock | `npm install` под Node 24 при необходимости | Согласование lockfile с движком | Оставить lock без изменений, если `npm ci` проходит |

## Risks / Trade-offs

- **[Risk]** У контрибьютора старая Node локально. **→** `engines` + README + `.nvmrc`.
- **[Trade-off]** Жёсткий pin `24.0.0` vs диапазон — предпочтительно **мажор** `24` без лишнего микропина, если CI использует последний патч 24.x.

## Migration Plan

1. Обновить workflow и файлы версии.
2. Локально: переключиться на Node 24, выполнить `npm ci` (или `npm install`).
3. Убедиться, что `npm run build` проходит.
4. Merge; проверить зелёный workflow.

## Open Questions

- Точный формат `engines` (`>=24.0.0` vs `^24`) — зафиксировать в задачах по согласованию с командой.
